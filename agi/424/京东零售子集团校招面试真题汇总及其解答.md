### 3.1 排序算法（Sorting Algorithms）

排序算法是算法基础中的一部分，常见的排序算法有冒泡排序（Bubble Sort）、选择排序（Selection Sort）、插入排序（Insertion Sort）、快速排序（Quick Sort）等。

#### Bubble Sort

Bubble Sort 是一种简单的排序算法，它重复地遍历要排序的数列，一次比较两个元素，如果他们的顺序错误就把他们交换过来。遍历数列的工作是重复地进行直到没有再需要交换，也就是说该数列已经排序完成。

#### Selection Sort

Selection Sort 是一种简单直观的排序算法，它的工作原理是每次从未排序的元素中找到最小（或最大）的元素，将其放到已排序序列的末尾。以此类推，直到所有元素均排序完毕。

#### Insertion Sort

Insertion Sort 插入排序是一种简单直观的排序算法，它的工作原理是通过构建有序序列，对于未排序数据，在已排序序列中从后向前扫描，找到相应位置并插入。

#### Quick Sort

Quick Sort 快速排序是一种高效的排序算法，其基本思想是通过一趟排序将待排序的记录分隔成独立的两部分，其中一部分记录的关键字均比另一部分的关键字小，则可分别对这两部分记录继续进行排序，以达到整个序列有序。

### 3.2 查找算法（Searching Algorithms）

查找算法是用于在数据结构中查找特定元素的方法。常见的查找算法有线性查找（Linear Search）、二分查找（Binary Search）等。

#### Linear Search

Linear Search 线性查找是从第一个元素开始，对线性表中的元素依次进行扫描，直到找到所要找的元素，或者直到扫描整个线性表。线性查找的时间复杂度为 O(n)，在最坏情况下，需要遍历整个数组。

#### Binary Search

Binary Search 二分查找是一种高效的查找算法，它将一个有序数组分成两部分，通过比较中间元素和目标元素的大小关系，来确定目标元素在数组的哪一部分，然后继续在相应部分进行二分查找。二分查找的时间复杂度为 O(log n)，在最好情况下，可以快速找到目标元素。

### 3.1 Sorting Algorithms

Sorting algorithms are a fundamental part of algorithmic fundamentals. Common sorting algorithms include Bubble Sort, Selection Sort, Insertion Sort, and Quick Sort.

#### Bubble Sort

Bubble Sort is a simple sorting algorithm that repeatedly traverses the array to be sorted, comparing and swapping adjacent elements if they are in the wrong order. This process is repeated until the array is sorted.

#### Selection Sort

Selection Sort is a straightforward sorting algorithm that works by finding the minimum (or maximum) element from the unsorted part of the array and placing it at the end of the sorted part. This process continues until all elements are sorted.

#### Insertion Sort

Insertion Sort is a simple and intuitive sorting algorithm that builds a sorted array one element at a time by comparing it with the elements before it and inserting it in the correct position.

#### Quick Sort

Quick Sort is an efficient sorting algorithm that works by selecting a 'pivot' element from the array and partitioning the other elements into two subsets, one with elements less than the pivot and the other with elements greater than the pivot. The algorithm then recursively sorts these subsets.

### 3.2 Searching Algorithms

Searching algorithms are used to find a specific element in a data structure. Common searching algorithms include Linear Search and Binary Search.

#### Linear Search

Linear Search scans through the array from the first element to the last, comparing each element with the target until a match is found, or the entire array is scanned. The time complexity of linear search is O(n), with worst-case performance occurring when the target element is at the end of the array.

#### Binary Search

Binary Search is an efficient searching algorithm that works on a sorted array. It repeatedly divides the search interval in half, comparing the middle element with the target, and narrowing down the interval accordingly. The time complexity of binary search is O(log n), providing fast performance even for large arrays.

$$

### 4.2 概率论公式（Probability Theory Formulas）

概率论是数学的一个分支，用于研究随机事件及其规律性。以下介绍几个常用的概率论公式：

1. 概率质量函数（Probability Quality Function）：

$$
P(A|B) = \frac{P(B|A)P(A)}{P(B)}
$$

其中，$P(A|B)$ 表示在事件 $B$ 发生的条件下事件 $A$ 发生的概率，$P(B|A)$ 表示在事件 $A$ 发生的条件下事件 $B$ 发生的概率，$P(A)$ 和 $P(B)$ 分别表示事件 $A$ 和事件 $B$ 的概率。

2. 贝叶斯定理（Bayes' Theorem）：

$$
P(A|B) = \frac{P(B|A)P(A)}{\sum_{i} P(B_i|A)P(A_i)}
$$

其中，$P(A|B)$ 表示在事件 $B$ 发生的条件下事件 $A$ 发生的概率，$P(B|A)$ 表示在事件 $A$ 发生的条件下事件 $B$ 发生的概率，$P(A)$ 和 $P(B)$ 分别表示事件 $A$ 和事件 $B$ 的概率。

#### Example

假设我们要判断一个产品是否合格，已知合格产品的概率为 0.9，不合格产品的概率为 0.1。在检测过程中，合格产品的检测合格率为 0.95，不合格产品的检测合格率为 0.2。

1. 计算在检测结果为合格的情况下，产品为合格的概率：

$$
P(\text{产品合格}|\text{检测结果合格}) = \frac{P(\text{检测结果合格}|\text{产品合格})P(\text{产品合格})}{P(\text{检测结果合格})}
$$

$$
P(\text{产品合格}|\text{检测结果合格}) = \frac{0.95 \times 0.9}{0.95 \times 0.9 + 0.2 \times 0.1} \approx 0.944
$$

2. 计算在检测结果为不合格的情况下，产品为不合格的概率：

$$
P(\text{产品不合格}|\text{检测结果不合格}) = \frac{P(\text{检测结果不合格}|\text{产品不合格})P(\text{产品不合格})}{P(\text{检测结果不合格})}
$$

$$
P(\text{产品不合格}|\text{检测结果不合格}) = \frac{0.2 \times 0.1}{0.95 \times 0.9 + 0.2 \times 0.1} \approx 0.056
$$

### 4.2 Probability Theory Formulas

Probability theory is a branch of mathematics that studies the patterns of random events. Here are several important probability formulas:

1. **Conditional Probability Quality Function**:

$$
P(A|B) = \frac{P(B|A)P(A)}{P(B)}
$$

where \( P(A|B) \) is the probability of event \( A \) occurring given that event \( B \) has occurred, \( P(B|A) \) is the probability of event \( B \) occurring given that event \( A \) has occurred, \( P(A) \) and \( P(B) \) are the probabilities of events \( A \) and \( B \), respectively.

2. **Bayes' Theorem**:

$$
P(A|B) = \frac{P(B|A)P(A)}{\sum_{i} P(B_i|A)P(A_i)}
$$

where \( P(A|B) \) is the probability of event \( A \) occurring given that event \( B \) has occurred, \( P(B|A) \) is the probability of event \( B \) occurring given that event \( A \) has occurred, \( P(A) \) and \( P(B) \) are the probabilities of events \( A \) and \( B \), respectively.

#### Example

Suppose we need to determine if a product is acceptable, given that the probability of an acceptable product is 0.9 and the probability of an unacceptable product is 0.1. During the inspection process, the acceptance rate for acceptable products is 0.95, and for unacceptable products, it is 0.2.

1. **Calculate the probability of an acceptable product given a positive test result**:

$$
P(\text{Product acceptable}|\text{Positive test result}) = \frac{P(\text{Positive test result}|\text{Product acceptable})P(\text{Product acceptable})}{P(\text{Positive test result})}
$$

$$
P(\text{Product acceptable}|\text{Positive test result}) = \frac{0.95 \times 0.9}{0.95 \times 0.9 + 0.2 \times 0.1} \approx 0.944
$$

2. **Calculate the probability of an unacceptable product given a negative test result**:

$$

$$

$$

### 4.3 Markov Chains

Markov Chains are mathematical models used to represent a sequence of possible events, where the probability of each event depends only on the state achieved in the previous event.

#### Formula

$$
P(X_n = x_n | X_{n-1} = x_{n-1}, ..., X_1 = x_1) = P(X_n = x_n | X_{n-1} = x_{n-1})
$$

This formula states that the probability of transitioning to a particular state at time \( n \) depends only on the state at time \( n-1 \), regardless of the sequence of events that led to it.

#### Example

Suppose we have a Markov Chain representing the state of a weather system, where \( S_0 \) is "sunny" and \( S_1 \) is "rainy". The transition matrix is:

$$
P =
\begin{bmatrix}
0.8 & 0.2 \\
0.3 & 0.7
\end{bmatrix}
$$

If today is sunny (\( S_0 \)), what is the probability of it being rainy tomorrow?

Using the Markov Chain formula:

$$
P(S_1 = \text{rainy} | S_0 = \text{sunny}) = P_{0,1} = 0.2
$$

So, the probability of it being rainy tomorrow is 0.2.

```python
class Stack:
    def __init__(self):

    def is_empty(self):
```

```python
        if not self.is_empty():
            return self.items[-1]
        else:
            raise IndexError("Peek from empty stack")

    def size(self):
        return len(self.items)
```

#### 5.3 Code Analysis

Let's break down the code step by step:

1. `class Stack:`: This line defines a new class named `Stack`.
2. `def __init__(self):`: The constructor initializes an empty list named `items` to store the stack elements.
3. `def is_empty(self):`: This method checks if the stack is empty by evaluating the length of the `items` list.
4. `def push(self, item):`: This method adds an element to the top of the stack.
5. `def pop(self):`: This method removes and returns the top element from the stack. If the stack is empty, it raises an `IndexError`.
6. `def peek(self):`: This method returns the top element from the stack without removing it. If the stack is empty, it raises an `IndexError`.
7. `def size(self):`: This method returns the number of elements in the stack.

#### 5.4 Running Results and Analysis

To test the functionality of the `Stack` class, we will run a series of tests:

```python

stack.push(3)

# Print the stack size
print("Stack size:", stack.size())  # Output: Stack size: 3

# Peek the top element
print("Top element:", stack.peek())  # Output: Top element: 3

# Pop elements from the stack
print("Popped element:", stack.pop())  # Output: Popped element: 3
print("Popped element:", stack.pop())  # Output: Popped element: 2

# Print the stack size again

```

The output of these tests demonstrates the basic operations of a stack: pushing and popping elements, checking the size, and peeking at the top element. The code successfully implements a stack using Python, showcasing how to create classes and define methods for managing data structures.
