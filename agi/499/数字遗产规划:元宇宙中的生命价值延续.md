### 4. 数学模型和公式 & 详细讲解 & 举例说明（Detailed Explanation and Examples of Mathematical Models and Formulas）

#### 4.1 数据加密与访问控制

在数字遗产规划中，数据加密和访问控制是非常关键的环节。以下是一些常用的数学模型和公式：

1. **数据加密模型**：
   - **对称加密**：使用相同的密钥进行加密和解密。公式为：
     $$ C = E_K(P) $$
     其中，\( C \) 是密文，\( P \) 是明文，\( K \) 是密钥。
   - **非对称加密**：使用一对密钥进行加密和解密。公式为：
     $$ C = E_K(P) $$
     其中，\( C \) 是密文，\( P \) 是明文，\( K \) 是公钥。
   - **哈希函数**：用于生成数据的唯一摘要。常见的哈希函数有MD5、SHA-256。公式为：
     $$ H(P) = Hash(P) $$
     其中，\( H \) 是哈希函数，\( P \) 是输入数据。

2. **访问控制模型**：
   - **访问控制列表（ACL）**：用于定义用户对数据的访问权限。公式为：
     $$ Access(P, U) = \{ permission | U \in UserSet \land P \in ResourceSet \} $$
     其中，\( Access \) 表示访问权限，\( P \) 表示资源，\( U \) 表示用户，\( UserSet \) 和 \( ResourceSet \) 分别表示用户集合和资源集合。

#### 4.2 示例说明

以下是一个简单的数据加密和访问控制的示例：

1. **数据加密示例**：

假设用户Alice想要加密一份重要文档，使用AES对称加密算法，密钥为K：

- 明文（P）：
  $$ P = "This is a secret document." $$
- 密文（C）：
  $$ C = E_K(P) $$

2. **访问控制示例**：

假设有如下用户和资源：

- 用户集合（UserSet）：
  $$ UserSet = \{ Alice, Bob, Carol \} $$
- 资源集合（ResourceSet）：
  $$ ResourceSet = \{ Document1, Document2, Document3 \} $$
- 访问控制列表（ACL）：
  $$ Access(P, U) = \{ "Alice": "read", "Bob": "write", "Carol": "none" \} $$

根据ACL，Alice可以读取Document1，Bob可以写入Document2，而Carol没有访问权限。

通过上述数学模型和公式，我们可以确保数字遗产在元宇宙中的安全性和访问控制。在实际应用中，这些模型和公式会根据具体需求进行复杂化和优化。

#### 4.1 Data Encryption and Access Control

In digital legacy planning, data encryption and access control are crucial aspects. Here are some common mathematical models and formulas used in this field:

1. **Data Encryption Models**:
   - **Symmetric Encryption**: Uses the same key for both encryption and decryption. The formula is:
     $$ C = E_K(P) $$
     where \( C \) is the ciphertext, \( P \) is the plaintext, and \( K \) is the key.
   - **Asymmetric Encryption**: Uses a pair of keys for encryption and decryption. The formula is:
     $$ C = E_K(P) $$
     where \( C \) is the ciphertext, \( P \) is the plaintext, and \( K \) is the public key.
   - **Hash Functions**: Used to generate a unique digest of data. Common hash functions include MD5 and SHA-256. The formula is:
     $$ H(P) = Hash(P) $$
     where \( H \) is the hash function and \( P \) is the input data.

2. **Access Control Models**:
   - **Access Control Lists (ACL)**: Used to define user access permissions for data. The formula is:
     $$ Access(P, U) = \{ permission | U \in UserSet \land P \in ResourceSet \} $$
     where \( Access \) represents access permissions, \( P \) represents a resource, \( U \) represents a user, \( UserSet \) and \( ResourceSet \) represent user and resource sets, respectively.

#### 4.2 Example Illustration

Here is a simple example of data encryption and access control:

1. **Data Encryption Example**:

Suppose user Alice wants to encrypt an important document using the AES symmetric encryption algorithm with key \( K \):

- Plaitext (\( P \)) :
  $$ P = "This is a secret document." $$
- Ciphertext (\( C \)) :
  $$ C = E_K(P) $$

2. **Access Control Example**:

Suppose we have the following users and resources:

- User set (\( UserSet \)) :
  $$ UserSet = \{ Alice, Bob, Carol \} $$
- Resource set (\( ResourceSet \)) :
  $$ ResourceSet = \{ Document1, Document2, Document3 \} $$
- Access control list (\( ACL \)) :
  $$ Access(P, U) = \{ "Alice": "read", "Bob": "write", "Carol": "none" \} $$

According to the \( ACL \), Alice has read access to Document1, Bob has write access to Document2, and Carol has no access permissions.

By using these mathematical models and formulas, we can ensure the security and access control of digital legacy in the Metaverse. In practical applications, these models and formulas are often complexified and optimized based on specific requirements.

        self.unconfirmed_transactions = []
        self.chain = []

    def create_genesis_block(self):
        genesis_block = {
            'previous_hash': 1,
            'index': 0,
            'transactions': []
        }
        hash = self.hash_block(genesis_block)
        genesis_block['hash'] = hash
        self.chain.append(genesis_block)

    def add_block(self, block):
        previous_block = self.chain[-1]
        block['previous_hash'] = previous_block['hash']
        hash = self.hash_block(block)
        block['hash'] = hash
        self.chain.append(block)

    def hash_block(self, block):
        block_string = json.dumps(block, sort_keys=True)
        return sha256(block_string.encode()).hexdigest()

    def set_access(self, owner, heir, access_permission):
        for block in self.chain:
            if block['owner'] == owner:
                block['heir'] = heir
                block['access_permission'] = access_permission
                return True
        return False

    def transfer_legacy(self, owner, heir):
        for block in self.chain:
            if block['owner'] == owner and block['heir'] == heir:
                self.set_access(owner, heir, 'transfer')
                return True
        return False
