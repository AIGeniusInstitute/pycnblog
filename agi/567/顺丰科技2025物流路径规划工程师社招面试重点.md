#### 3.1 Dijkstra算法

Dijkstra算法是一种经典的单源最短路径算法，用于求解给定的加权图中从单一源点出发到其他所有节点的最短路径。Dijkstra算法的基本原理是通过不断更新节点的最短路径估计值，直到找到所有节点的最短路径。

##### 3.1.1 算法原理

1. 初始化：设置源点s的初始距离为0，其他节点的距离为无穷大。将源点s加入已访问节点集，其他节点加入未访问节点集。

2. 选取未访问节点中距离最小的节点作为当前节点。

3. 更新当前节点的邻居节点的距离：对于当前节点的每个邻居节点v，计算从源点s到v的最短路径，如果新路径比当前已知的路径更短，则更新v的距离值。

4. 重复步骤2和3，直到所有节点都被访问过。

##### 3.1.2 具体操作步骤

1. 初始化：创建一个长度为n的数组dist，用于存储从源点s到每个节点的最短路径距离，初始化为dist[s] = 0，其余为无穷大。创建一个布尔数组visited，用于标记节点是否被访问，初始化为false。

2. 选择未访问节点中距离最小的节点作为当前节点。

3. 对于当前节点的每个邻居节点v：

    a. 如果v未被访问且从s到v的新路径比当前已知的路径更短，则更新dist[v]。

    b. 标记v为已访问。

4. 重复步骤2和3，直到所有节点都被访问过。

5. 输出从源点s到每个节点的最短路径距离。

#### 3.2 A*算法

A*算法是一种启发式搜索算法，用于求解给定加权图中从源点s到目标点t的最短路径。A*算法的基本原理是基于启发式函数来估算从源点到目标点的距离，并选择估计距离最小的节点进行扩展。

##### 3.2.1 算法原理

1. 初始化：设置源点s的初始距离为0，目标点t的初始距离为无穷大。将源点s加入已访问节点集，目标点t加入未访问节点集。

2. 选择未访问节点中F值（启发式函数值 + G值，即估计距离）最小的节点作为当前节点。

3. 对于当前节点的每个邻居节点v：

    a. 计算从当前节点到邻居节点v的G值（实际距离）。

    b. 如果v未访问或新的G值更小，则更新v的F值和前驱节点。

    c. 如果v是目标点，则结束搜索。

4. 重复步骤2和3，直到找到目标点或所有节点都被访问过。

##### 3.2.2 具体操作步骤

1. 初始化：创建两个优先级队列openSet和closedSet，分别用于存储未访问节点和已访问节点。创建一个长度为n的数组dist，用于存储从源点s到每个节点的最短路径距离，初始化为dist[s] = 0，其余为无穷大。创建一个布尔数组visited，用于标记节点是否被访问，初始化为false。

2. 将源点s加入openSet。

3. 选择openSet中F值最小的节点作为当前节点。

4. 将当前节点从openSet移动到closedSet。

5. 对于当前节点的每个邻居节点v：

    a. 如果v在closedSet中，则跳过。

    b. 计算从当前节点到邻居节点v的G值。

    c. 如果v未访问或新的G值更小，则更新v的F值和前驱节点，并将v加入openSet。

    d. 如果v是目标点，则找到最短路径并结束搜索。

6. 重复步骤3至5，直到找到目标点或openSet为空。

7. 输出从源点s到目标点t的最短路径。

### 3. Core Algorithm Principles and Specific Operational Steps
#### 3.1 Dijkstra Algorithm
Dijkstra algorithm is a classical single-source shortest path algorithm used to solve the shortest path problem from a single source node to all other nodes in a given weighted graph. The basic principle of Dijkstra algorithm is to continuously update the shortest path estimate of each node until the shortest path to all nodes is found.

##### 3.1.1 Algorithm Principles
1. Initialization: Set the initial distance of the source node s to 0 and the distance of other nodes to infinity. Add the source node s to the visited set and other nodes to the unvisited set.
2. Select the node with the smallest distance from the unvisited nodes as the current node.
3. Update the distance of the neighbor nodes of the current node: For each neighbor node v of the current node, calculate the shortest path from the source node s to v. If the new path is shorter than the known path, update v's distance.
4. Repeat steps 2 and 3 until all nodes have been visited.

##### 3.1.2 Specific Operational Steps
1. Initialization: Create an array dist of length n to store the shortest path distance from the source node s to each node, initialized as dist[s] = 0, others as infinity. Create a boolean array visited to mark whether a node has been visited, initialized as false.
2. Select the unvisited node with the smallest distance as the current node.
3. For each neighbor node v of the current node:
   a. If v is unvisited and the new path from s to v is shorter than the known path, update dist[v].
   b. Mark v as visited.
4. Repeat steps 2 and 3 until all nodes have been visited.
5. Output the shortest path distance from the source node s to each node.

#### 3.2 A* Algorithm
A* algorithm is a heuristic search algorithm used to solve the shortest path problem from a single source node to a target node in a given weighted graph. The basic principle of A* algorithm is to use a heuristic function to estimate the distance from the source node to the target node and select the node with the smallest estimated distance for expansion.

##### 3.2.1 Algorithm Principles
1. Initialization: Set the initial distance of the source node s to 0 and the distance of the target node t to infinity. Add the source node s to the visited set and the target node t to the unvisited set.
2. Select the node with the smallest F-value (heuristic function value + G-value, i.e., estimated distance) from the unvisited nodes as the current node.
3. For each neighbor node v of the current node:
   a. Calculate the G-value (actual distance) from the current node to the neighbor node v.
   b. If v is unvisited or the new G-value is smaller, update v's F-value and predecessor node.
   c. If v is the target node, end the search.
4. Repeat steps 2 and 3 until the target node is found or all nodes have been visited.

##### 3.2.2 Specific Operational Steps
1. Initialization: Create two priority queues openSet and closedSet to store the unvisited nodes and visited nodes, respectively. Create an array dist of length n to store the shortest path distance from the source node s to each node, initialized as dist[s] = 0, others as infinity. Create a boolean array visited to mark whether a node has been visited, initialized as false.
2. Add the source node s to openSet.
3. Select the node with the smallest F-value from openSet as the current node.
4. Move the current node from openSet to closedSet.
5. For each neighbor node v of the current node:
   a. If v is in closedSet, skip.
   b. Calculate the G-value from the current node to the neighbor node v.
   c. If v is unvisited or the new G-value is smaller, update v's F-value and predecessor node, and add v to openSet.
   d. If v is the target node, find the shortest path and end the search.
6. Repeat steps 3 to 5 until the target node is found or openSet is empty.
7. Output the shortest path from the source node s to the target node t.

#### 4.1 基本概念

在物流路径规划中，常用的数学模型包括距离模型、时间模型、成本模型等。

##### 4.1.1 距离模型

距离模型是物流路径规划中最基本的模型，用于计算两个节点之间的距离。常用的距离模型包括欧几里得距离、曼哈顿距离等。

- **欧几里得距离**：两个节点 \( i \) 和 \( j \) 之间的欧几里得距离公式为：
  \[ d_{ij} = \sqrt{(x_i - x_j)^2 + (y_i - y_j)^2} \]
  其中，\( (x_i, y_i) \) 和 \( (x_j, y_j) \) 分别是节点 \( i \) 和 \( j \) 的坐标。

- **曼哈顿距离**：两个节点 \( i \) 和 \( j \) 之间的曼哈顿距离公式为：
  \[ d_{ij} = |x_i - x_j| + |y_i - y_j| \]

##### 4.1.2 时间模型

时间模型用于计算两个节点之间的时间距离。时间模型的计算通常依赖于交通流量、道路状况等因素。一个简单的时间模型是使用公式：
\[ t_{ij} = \frac{d_{ij}}{v_i} \]
其中，\( t_{ij} \) 是从节点 \( i \) 到节点 \( j \) 的行驶时间，\( d_{ij} \) 是两个节点之间的距离，\( v_i \) 是道路上的平均速度。

##### 4.1.3 成本模型

成本模型用于计算物流路径的总成本，包括运输成本、等待成本、罚款成本等。一个简单的成本模型是使用公式：
\[ C_{ij} = c_1 \cdot d_{ij} + c_2 \cdot t_{ij} + c_3 \cdot \text{罚分} \]

### 4. Mathematical Models and Formulas & Detailed Explanation & Examples
#### 4.1 Basic Concepts
In logistics route planning, common mathematical models include distance models, time models, and cost models.

##### 4.1.1 Distance Model
The distance model is the most basic model in logistics route planning, used to calculate the distance between two nodes. Common distance models include Euclidean distance and Manhattan distance.

- **Euclidean Distance**: The Euclidean distance between nodes \( i \) and \( j \) is given by the formula:
  \[ d_{ij} = \sqrt{(x_i - x_j)^2 + (y_i - y_j)^2} \]
  where \( (x_i, y_i) \) and \( (x_j, y_j) \) are the coordinates of nodes \( i \) and \( j \), respectively.

- **Manhattan Distance**: The Manhattan distance between nodes \( i \) and \( j \) is given by the formula:
  \[ d_{ij} = |x_i - x_j| + |y_i - y_j| \]

##### 4.1.2 Time Model
The time model is used to calculate the time distance between two nodes. The calculation of the time model often depends on traffic flow, road conditions, and other factors. A simple time model is given by the formula:
\[ t_{ij} = \frac{d_{ij}}{v_i} \]
where \( t_{ij} \) is the driving time from node \( i \) to node \( j \), \( d_{ij} \) is the distance between nodes \( i \) and \( j \), and \( v_i \) is the average speed on the road.

##### 4.1.3 Cost Model
The cost model is used to calculate the total cost of a logistics route, including transportation costs, waiting costs, and penalty costs. A simple cost model is given by the formula:
\[ C_{ij} = c_1 \cdot d_{ij} + c_2 \cdot t_{ij} + c_3 \cdot \text{penalty} \]
where \( C_{ij} \) is the total cost from node \( i \) to node \( j \), \( c_1 \), \( c_2 \), and \( c_3 \) are the proportion coefficients for transportation cost, time cost, and penalty cost, respectively.

from ortools.constraint_solver import pywrapcp

def create_data_model():
    data = {}
    data['distance_matrix'] = [
        [0, 5, 8, 10],  # 起点A到各节点的距离
        [5, 0, 3, 6],  # 节点B到各节点的距离
        [8, 3, 0, 2],  # 节点C到各节点的距离
        [10, 6, 2, 0]  # 节点D到各节点的距离
    ]
    data['depot'] = 0  # 起点A的索引
    data['vehicle_count'] = 2  # 车辆数
    data['route_list'] = []  # 路线列表
    return data

def print_solution(data, manager, routing, solution):
    print('Objective: minimal distance = {} miles'.format(solution.ObjectiveValue()))
    total_distance = 0
    for path_index in range(1, data['vehicle_count'] + 1):
        index = routing.Start(path_index)
        route_distance = 0
        while not routing.IsEnd(index):
            node_index = manager.IndexToNode(index)
            route_distance += routing.GetArcCostForVehicle(index, index)
            index = solution.Value(routing.NextVar(index))
        data['route_list'].append(route_distance)
        total_distance += route_distance
        print('Route for vehicle {} ({})'.format(path_index, sum(data['distance_matrix'][node_index][node_index] for node_index in routing.GetRouteDistance(path_index))))
        print('  Distance = {} miles'.format(route_distance))
    print('Total distance = {} miles'.format(total_distance))

def main():
    data = create_data_model()
    manager = pywrapcp.RoutingIndexManager(len(data['distance_matrix']), data['vehicle_count'], data['depot'])
    routing = pywrapcp.RoutingModel(manager)
    search_parameters = pywrapcp.RoutingModel.DefaultSearchParameters()
    search_parameters.first_solution_strategy = routing_enums_pb2.FirstSolutionStrategy.Value('PATH_CHEAPening')
    solution = routing.SolveFromSolution(search_parameters, routing.FirstSolution())
    print_solution(data, manager, routing, solution)

if __name__ == '__main__':
    main()

Dijkstra算法是一种单源最短路径算法，用于求解给定加权图中从单一源点出发到其他所有节点的最短路径。它通过不断更新节点的最短路径估计值，直到找到所有节点的最短路径。

#### 9.4 什么是A*算法？

A*算法是一种启发式搜索算法，用于求解给定加权图中从源点s到目标点t的最短路径。它利用启发式函数来估算从源点到目标点的距离，并选择估计距离最小的节点进行扩展。

#### 9.3 What is Dijkstra algorithm?

Dijkstra algorithm is a single-source shortest path algorithm used to solve the shortest path problem from a single source node to all other nodes in a given weighted graph. It continuously updates the shortest path estimate of each node until the shortest path to all nodes is found.

#### 9.4 What is A* algorithm?

A* algorithm is a heuristic search algorithm used to solve the shortest path problem from a single source node to a target node in a given weighted graph. It uses a heuristic function to estimate the distance from the source node to the target node and selects the node with the smallest estimated distance for expansion.
